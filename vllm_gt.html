01. LLM 서빙의 한계

기존 KV 캐시 시스템의 메모리 병목 현상

KV Cache: 메모리 병목의 원인

  • 거대한 메모리 요구량: 수천 개의 토큰에 대한 Key/Value 값을 GPU에 유지해야 함
  • 파편화(Fragmentation): 정적 할당 방식으로 인해 최대 길이를 예약하여 60~80%의 메모리가 낭비됨
  • 낮은 Throughput: 메모리 낭비로 인해 한 번에 처리 가능한 요청 수(Batch Size)가 극히 제한됨

PagedAttention 기술

OS의 가상 메모리 페이징 기법을 LLM에 도입했습니다.

KV 캐시를 고정된 크기의 '블록(Block)'으로 분할하여 물리적 메모리의 불연속적인 위치에 동적으로 할당합니다.

결과: 파편화 0%에 근접, 2-4배 많은 요청 처리 가능

메모리 관리의 혁신: Logical to Physical

Logical Blocks

사용자의 요청은 논리적으로 연속된 토큰들의 배열로 보입니다. 각 토큰 그룹은 논리 블록 인덱스로 관리됩니다.

Physical Blocks

실제 GPU 메모리는 블록 매니저가 관리하는 불연속적 위치에 존재합니다. 매핑 테이블을 통해 지연 할당(Lazy Allocation)을 수행합니다.

Continuous Batching (반복 수준 스케줄링)

Iteration-level

배치의 모든 요청이 끝날 때까지 기다리지 않고, 매 토큰 생성 반복마다 새로운 요청을 삽입합니다.

Zero Idle Time

먼저 끝난 요청이 자리를 비우면 즉시 다음 대기 요청이 시작되어 GPU 활용률을 극대화합니다.

Throughput Boost

정적 배치(Static Batching) 대비 수배 이상의 처리량 향상을 보장합니다.

메모리 낭비의 극적인 감소

PagedAttention은 기존의 정적 할당 방식에서 발생하던 내부 파편화(Internal Fragmentation)와 예약 낭비를 거의 완벽하게 제거합니다. 동일 사양의 GPU에서 최대 4배 더 많은 모델 인스턴스나 배치를 운용할 수 있습니다.

CUDA 최적화 및 추가 기능

  • Custom CUDA Kernels: PagedAttention 연산을 위해 최적화된 독자적인 커널 구현
  • CUDA Graph: 모델 실행 오버헤드를 줄이기 위한 연산 그래프 캡처 지원
  • Prefix Caching: 동일한 프롬프트(System Prompt 등)를 공유할 경우 메모리 중복 사용 방지
  • Quantization Stack: AWQ, GPTQ, SqueezeLLM 등 최신 양자화 기술과 FP8 KV 캐시 완벽 지원
